# Validation: T-LARS/T-LASSO vs. R `tlars` Path Equivalence (Python)

## Purpose

Final forward-selection path-equivalence test comparing the CRAN `tlars`
package against the Python-binding `TLARS`/`TLASSO`/`TENET`/`TENETAug`
solvers, using the same sparse factor-model data as the SPCA demos. Python
mirror of the cpp comparator of the same name.

Five solver runs are checked against the matching CRAN reference:

| Solver run | Reference files |
|---|---|
| `TLARS_Solver` | `r_lar_*` |
| `TLASSO_Solver` | `r_lasso_*` |
| `TENET_Solver` (Gram EN) | `r_en_*` |
| `TENETAug_Solver` (default LASSO inner) | `r_en_*` |
| `TENETAug_Solver` (`use_lars_inner=True`) | `r_enlar_*` |

Binding note: unlike the cpp comparator, the Python binding's `getActions()`
already returns signed 1-based actions in the R `lars` convention, so no index
shift is needed when comparing action sequences.

---

## Running

This program does **not** generate its own data: it reads R-produced reference
CSVs (`meta.csv`, `Xn_<i>.csv`, `Dn_<i>.csv`, `y_<i>.csv`, and the `r_lar_*`,
`r_lasso_*`, `r_en_*`, `r_enlar_*` beta/action files) generated by
[`R/tsolvers/validation/validation_ts_02_tlars_tlasso_rcompare/demo_ts_compare_tlars_tlasso.R`](../../../../R/tsolvers/validation/validation_ts_02_tlars_tlasso_rcompare/demo_ts_compare_tlars_tlasso.R).

By default it looks for them in
`R/tsolvers/validation/validation_ts_02_tlars_tlasso_rcompare/rdump_tlars/`.
Override the location with the optional `--dir <path>` flag:

```bash
# uses the default reference directory
.venv/bin/python Python/tsolvers/validation/validation_ts_02_tlars_tlasso_rcompare/validation_ts_02_tlars_tlasso_rcompare.py

# or point it at an explicit reference-data directory
.venv/bin/python Python/tsolvers/validation/validation_ts_02_tlars_tlasso_rcompare/validation_ts_02_tlars_tlasso_rcompare.py --dir /path/to/rdump_tlars
```

The regression gate is reflected in the exit code: `0` on PASS, `1` on FAIL.

## What to Look For

- Exact (or near machine-precision) agreement between Python and R selection paths and coefficients
- Any divergence points (e.g., tie-breaking differences) should be flagged and documented
- Because `p + L > n`, plain LARS/LASSO comparisons are capped at the full-rank region (`n` columns); the EN runs operate on the full-rank augmented system and are not capped
