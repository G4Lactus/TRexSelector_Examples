# Demo 01: Single-Run T-Rex Selector (Python)

## Purpose

Demonstrate basic T-Rex selector usage and output format via two contrasting
single-run scenarios using the TLARS solver. Python port of
`cpp/trex_selector_methods/trex/demo_trex_01_single_run` and the matching R demo.

## DGP / Data

Gaussian i.i.d. design generated by the shared `dgp_gauss_snr`:

- Fixed true support (0-based): `{27, 149, 43, 128, 42, 4}` (`s = 6`)
- Fixed coefficients `beta_j = 1` (`rnd_coef=False`; as in C++, the `rnd_coef`
  path selects a fixed hardcoded vector that mimics random coefficients — it
  is not a random draw)
- SNR = 1.0, tFDR = 0.1
- High-dimensional (`p > n`): `n = 150`, `p = 300` — run first, as in C++
- Low-dimensional (`n > p`): `n = 5000`, `p = 1000`
- Seeds mirror C++ structurally: data seed `58`, selector seed `42`. The RNG
  streams differ across languages, so the drawn data (and hence the exact
  selection) are not bit-identical to the C++ output.

## What It Computes

`part_1_single_run()` runs the high-dimensional case first, then the
low-dimensional one. Each run builds a `tsn.TRexSelector` with a `STANDARD`
L-loop control (`K = 20`, `max_dummy_multiplier = 10`, `use_max_T_stop = True`,
`tloop_stagnation_stop = False`) and calls `select()`. Results are printed via
`print_single_run_result`: a data/calibration header, then the C++-style
selection block — selected indices, FDP/TPP, `phi_prime` and `phi_mat`
(printed **sparsely**, nonzero entries with their indices, mirroring
`trex_sim::print_sparse_vector` / `print_sparse_matrix`), `fdp_hat_mat`,
`r_mat`, and `voting_grid`.

This is a single-seed demo, not a Monte Carlo study.

## Imports

The demo inserts both its own folder and the parent suite dir onto `sys.path`
at the top of the file, so it can import the shared modules `dgp_gauss_snr`,
`trex_helpers`, and `trex_sim_common`. The selector layer is used via the root
alias `import trex_selector_neo as tsn` (`tsn.TRexSelector`).

## Output Files

Dual console + file output (mirrors the C++ `print_dual` pattern). One `.txt`
per scenario in `simulation_results/data/`, stem `d01_trex_basic_n{n}_p{p}.txt`:

- `d01_trex_basic_n150_p300.txt` (high-dimensional)
- `d01_trex_basic_n5000_p1000.txt` (low-dimensional)

## Running

```bash
python Python/trex_selector_methods/trex/demo_trex_01_single_run/demo_trex_01_single_run.py
```
