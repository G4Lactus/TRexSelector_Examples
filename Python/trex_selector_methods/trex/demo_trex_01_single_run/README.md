# Demo 01: Single-Run T-Rex Selector (Python)

## Purpose

Demonstrate basic T-Rex selector usage and output format via two contrasting
single-run scenarios using the TLARS solver. Python port of
`cpp/trex_selector_methods/trex/demo_trex_01_single_run` and the matching R demo.

## DGP / Data

Gaussian i.i.d. design generated by the shared `dgp_gauss_snr`:

- Fixed true support (0-based): `{27, 149, 43, 128, 42, 4}` (`s = 6`)
- Fixed coefficients `beta_j = 1` (`amplitude = 1.0`)
- SNR = 1.0, data/selector seed = 4231, tFDR = 0.1
- Low-dimensional (`n > p`): `n = 5000`, `p = 1000`
- High-dimensional (`p > n`): `n = 150`, `p = 300`

## What It Computes

`part_1_single_run()` runs the low-dimensional case first, then the
high-dimensional one. Each run builds a `trex_selector_neo.TRexSelector` with a
`STANDARD`/`HCONCAT` control (`K = 20`, `max_dummy_multiplier = 10`,
`lloop_strategy = "HCONCAT"`, `tloop_stagnation_stop = False`) and calls
`select()`. Results are printed via `print_single_run_result`, reporting the
selected indices, FDP/TPP, and the `phi_prime`, `phi_mat`, `fdp_hat_mat`,
`r_mat`, and `voting_grid` diagnostics.

This is a single-seed demo, not a Monte Carlo study. It is **console-only** —
no result files are written.

## Imports

The demo inserts both its own folder and the parent suite dir onto `sys.path`
at the top of the file, so it can import the shared modules `dgp_gauss_snr`,
`trex_sim_common`, and `trex_helpers`. The package itself imports as
`trex_selector_neo`.

## Running

```bash
.venv/bin/python Python/trex_selector_methods/trex/demo_trex_01_single_run/demo_trex_01_single_run.py
```
